# 雀胡弈 (Jokhan)

基于麻将元素的 Roguelike 卡牌构筑游戏，核心玩法借鉴《Balatro（小丑牌）》的倍率叠加+闯关循环。

![游戏UI素材](/docs/icons.png)

## 核心对标

| Balatro | 雀胡弈 |
|---------|--------|
| 扑克牌组（52张） | 麻将牌（万/条/筒 + 字牌，136张） |
| 5张手牌打出牌型 | **5张手牌**，打出对子/顺子/刻子 |
| 小丑牌（倍率/特效） | **雀符**（玉佩/符咒样式，8槽） |
| 塔罗牌/星球牌 | **秘具**（一次性消耗道具，5槽） |
| 筹码 × 倍率 | **牌型基础番 × 雀符倍率** |
| 盲注挑战（目标分数） | 关卡目标分数，有限出牌/弃牌次数 |

## 核心玩法

- **5张手牌**：选择2-3张组成对子/顺子/刻子得分，打出后补满5张
- **对子**（2张相同）：1番 ×1.0 | **顺子**（3张同花连续）：2番 ×1.5 | **刻子**（3张相同）：3番 ×2.0
- **雀符**：最多携带8枚，提供番数加成和倍率放大，可叠加实现指数级爆分
- **秘具**：每局最多5个，一次性消耗，应急改变战局
- **宝牌系统**：牌山中随机标记宝牌，打出额外加分
- **每关有限出牌次数和弃牌次数**，需在耗尽前达到目标分数

## 关卡体系

1. **初阶坊市** — 新手关，目标400分，5次出牌，3次弃牌
2. **四方雀馆** — 目标800分，4出牌，3弃牌
3. **庄位试炼** — 目标1500分，4出牌，2弃牌
4. **四境天阙** — 目标3000分，3出牌，2弃牌

## 得分公式

```
单次出牌得分 = 100 × (牌型基础番 + 雀符固定加成) × 雀符倍率 × 牌型倍率
累计得分 ≥ 目标分数 → 通关
```

## 使用说明

### 环境要求

- Node.js >= 18
- npm >= 9

### 安装与运行

```bash
# 1. 克隆项目
git clone <repo-url>
cd Jokhan

# 2. 安装依赖
npm install

# 3. 开发模式（浏览器运行，热更新）
npm run dev

# 4. 桌面模式（Electron 窗口）
npx electron .
```

启动后在浏览器打开 `http://localhost:5173`，或在 Electron 窗口中直接游玩。

### 操作指南

| 操作 | 方法 |
|------|------|
| 选择手牌 | 点击牌面，可选 2~3 张（高亮升起 + 金框） |
| 出牌得分 | 选好有效牌型后，点击底栏「出牌」 |
| 弃牌换牌 | 选中要弃的牌，点击底栏「弃牌」 |
| 整理手牌 | 点击底栏「排序」，按花色数字排列 |
| 使用秘具 | 点击右侧秘具面板中的道具 |

### 牌型说明

| 牌型 | 组成 | 基础番 | 倍率 |
|------|------|--------|------|
| 对子 | 2张完全相同 | 1番 | ×1.0 |
| 顺子 | 3张同花色连续数字（如 1万2万3万） | 2番 | ×1.5 |
| 刻子 | 3张完全相同 | 3番 | ×2.0 |

### 打包发布

```bash
npm run build        # 生产构建
npm run pack:mac     # 打包 macOS DMG 安装包
npm run pack:win     # 打包 Windows EXE 安装包
```

打包产物在 `release/` 目录。

## 技术栈

| 层 | 技术 | 用途 |
|----|------|------|
| 桌面容器 | Electron | 跨平台，打包 DMG/EXE |
| 游戏引擎 | Phaser 3 | 2D 渲染、场景管理、动画 |
| 语言 | TypeScript | 类型安全 |
| 打包 | electron-builder | 一键出 macOS + Windows 安装包 |
| 构建 | Vite | 快速开发 HMR |

## 项目结构

```
src/
├── main/          Electron 主进程
├── preload/       预加载脚本
├── shared/        共享类型定义
└── renderer/
    ├── index.ts   Phaser 入口
    ├── scenes/    场景（Boot/Menu/Game/Settlement）
    ├── systems/   游戏逻辑（牌库/手牌/组合/得分/雀符/秘具）
    └── ui/        视觉主题
docs/              需求文档 + UI素材
```
